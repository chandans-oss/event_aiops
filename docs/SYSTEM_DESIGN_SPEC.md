# AIOps Event Analytics: System Design & Operational Specification

## 1. Overview
This document provides a detailed technical breakdown of the AIOps Event Analytics platform, structured around its core functional modules as represented in the operational dashboard's navigation system. The platform is designed to provide end-to-end observability, from raw event ingestion to ML-driven root cause analysis (RCA).

---

## 2. Navigation Architecture (`LeftSidebar.tsx`)
The application utilizes a sophisticated, stateful sidebar that orchestrates the user's journey across different operational layers.

### Technical Implementation:
- **Dynamic State Management**: Uses a combination of `useState` and `useEffect` to track menu expansion (`openMenus`) and collapsed states. It automatically expands parent categories based on the current `browser.location.path`.
- **Active State Tracking**: Leverages `react-router-dom`’s `NavLink` and `useLocation` to apply high-visibility styling to the active route, ensuring visual orientation.
- **Components**: Built using **Radix UI Collapsible** primitives for accessible, smooth accordion transitions and **Lucide-React** for semantic iconography.

---

## 3. Operational Layer Specifications

### 3.1 Dashboards Module (The Intelligence Hub)
This layer provides high-level visualization of infrastructure health and predictive insights.
- **AIOps Dashboard (`/`)**: 
  - **Component**: `AnalyticsDashboard`
  - **Purpose**: Real-time overview of Root Cause Insights, SLA Breach risk, and Capacity forecasting.
- **Topology Dashboard (`/dashboard/prediction`)**: 
  - **Component**: `PredictionDashboard`
  - **Purpose**: Maps logical and physical dependencies between network assets to visualize how alerts propagate through the infrastructure.
- **ROI/KPI Dashboard (`/dashboard/kpi`)**: 
  - **Component**: `KpiDashboard`
  - **Purpose**: Metrics-driven view focusing on the economic value generated by the platform (e.g., MTTR reduction, automation rate).

### 3.2 Events Module (The Operational Core)
The primary interface for NOC operators to interact with incoming telemetry.
- **Events View (`/events`)**: 
  - **Component**: `EventsPage`
  - **Technical Detail**: Implements high-performance virtualization for large event lists and real-time filtering/sorting of device-specific alarms.

### 3.3 Admin Module (The Optimization Layer)
Used by administrators to fine-tune the AI engines and correlation logic.
- **Configuration (`/admin`)**: 
  - **Component**: `AdminPage`
  - **Purpose**: System-wide settings and general platform configuration.
- **Correlation Patterns (`/correlation`)**: 
  - **Component**: `CorrelationPage`
  - **Purpose**: Management of human-in-the-loop correlation rules and behavioral pattern definitions.
- **Training (`/pattern-prediction/training`)**: 
  - **Component**: `TrainingLovelablePage`
  - **Purpose**: UI for configuring and launching ML training jobs, selecting algorithms (XGBoost, Random Forest), and defining training windows.
- **Results (`/pattern-prediction/results`)**: 
  - **Component**: `LovelableResultsPage`
  - **Purpose**: Detailed analytical reporting on historical training runs, including precision/recall metrics.

### 3.4 Playground Module (The Innovation Layer)
Advanced tools for testing and refining AIOps strategies.
- **RCA Playground (`/playground/rca`)**: 
  - **Component**: `RCAPlaygroundPage`
  - **Purpose**: A multi-step testing environment (Retriever -> Correlator -> Reporter) to validate RCA logic against historical event clusters.
- **Event Deduplication (`/event-processing/deduplication`)**: 
  - **Component**: `DeduplicationPage`
  - **Purpose**: Specialized logic for identifying and merging redundant alarms based on signature matching and temporal proximity.

## 4. Technical Stack Summary
- **Frontend Framework**: React 18 with functional components and specialized hooks.
- **Styling**: Tailwind CSS (Atomic CSS) + Custom CSS Variables for theme consistency.
- **Navigation**: React Router DOM (v6) with declarative route definitions in `App.tsx`.
- **Icons**: Lucide-React for highly recognizable, professional iconography.
